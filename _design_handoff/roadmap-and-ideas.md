# NerdDen — Roadmap & Product Ideas

Стратегические предложения по развитию платформы. Черновик для последующей реализации через Claude Code.
Северная звезда продукта: **AI генерирует игровой контент по любой теме**. Всё ниже подчинено этому.

---

## Часть 1. Улучшения платформы (приоритет сверху вниз)

### 1. Единый домашний хаб / home (САМОЕ НУЖНОЕ — делаем первым)
Сейчас у каждой игры своё лобби, но нет главной страницы, связывающей их.
- **Витрина трёх игр** (Sudoku · Crosswords · Alias) с маскотами.
- **Daily Den** — по одной задаче из каждой игры на день; закрытие продлевает общую серию.
- **Continue** — продолжить начатое во всех играх сразу.
- **Сквозной статус** — уровень/XP аккаунта, серия дней, быстрый CTA «Generate with AI».
- Экран-концентратор навигации; топбар-переключатель игр уже готов в макетах.

### 2. Сквозная (кросс-игровая) прогрессия
- Уровень/XP уже есть в Sudoku — сделать их **общими на аккаунт**.
- **Маскоты — коллекционные.** Сова/Шляпник в профиле уже показаны «под замком» — готовый крючок: разблокировка за достижения в соответствующей игре.
- Достижения и лидерборды — свести к аккаунту, а не к игре.

### 3. Шеринг сгенерированного контента (вирусный крючок)
- Кроссворд уже открывается по `/crossword/[id]` — превратить в фичу: «сгенерировал кроссворд про биологию → отправил ссылку/код другу».
- AI-генерация по теме — главный дифференциатор, идеально годится для шеринга.
- Расшарить набор слов Alias / пазл Sudoku тем же механизмом.

### 4. Асинхронные вызовы друзей
- Не только реалтайм-дуэль: «решишь мой набор быстрее?» с push-уведомлениями.
- Переиспользует competitive-инфраструктуру, но без требования «оба онлайн».

### 5. Онбординг и настройки
- Оба упоминаются в PLAN, но не спроектированы. Нужны экраны первого входа и settings (язык, тема, звук, доступность).

### 6. Доступность (a11y)
- Маркеры красный/зелёный на доске Sudoku могут сливаться у дальтоников — **colorblind-safe палитра** (или паттерны/формы, не только цвет).
- Довести переключатель **тёмной темы** (палитра уже в tokens.css).
- Хит-таргеты ≥44px (в макетах соблюдено), клавиатурная навигация.

### 7. PWA / офлайн
- Головоломки идеальны офлайн; синхронизация прогресса при возврате сети.
- Устанавливаемое приложение, кэш ассетов и текущей игры.

## Часть 2. Какие игры добавлять

### Вывод по шахматам: НЕ добавлять
- ДНК платформы — «AI генерирует контент по теме». В шахматах генерировать нечего — суперсила не работает.
- Рынок занят намертво (lichess, chess.com); крафт-эстетика и маскоты там чужеродны.

### Две опоры, которые стоит углублять

**Опора A — Логика / сетка** (вотчина Судоку-маньяка; переиспользует PixiJS-доску):
- **Нонограммы (Picross / японские кроссворды)** — визуально яркие, отлично ложатся на крафт-стиль.
- **Какуро** — «судоку с суммами», та же механика ввода.

**Опора B — Слова / компания** (сова + шляпник; переиспользует AI-генерацию + WS-комнаты Alias):
- **Викторина / квиз по теме** — сильнейший фит: AI генерит вопросы по любой теме, комнаты как в Alias. Почти бесплатно по инфраструктуре. **Рекомендуемый следующий шаг №1.**
- **Codenames-подобная** командная игра — прямое переиспользование команд+комнат Alias и AI-сетки слов.
- **Word Search / поиск слов** — AI даёт тематический список → сетка; переиспользует генерацию слов кроссворда.

Каждая новая игра = новый маскот (система «маскот на игру» уже заложена в дизайн-системе).

### Рекомендация по последовательности
1. **Единый домашний хаб** (в работе).
2. **Викторина по теме** (опора B) — максимальный фит с AI-суперсилой, минимум новой инфраструктуры.
3. **Нонограммы** (опора A) — визуальный хит, переиспользует доску.
4. Дальше — по данным: Codenames-like / Какуро / Word Search.

Держать AI-генерацию по теме как северную звезду; шахматы не трогать.
